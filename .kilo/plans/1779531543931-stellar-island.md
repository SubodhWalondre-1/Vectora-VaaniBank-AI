# AI Suggestion System Analysis

## Overview

The AI suggestion system in VaaniBank-AI provides real-time, context-aware response suggestions to bank staff during customer interactions. It combines Speech-to-Text (STT), Large Language Model (LLM) processing, and real-time WebSocket communication.

## Complete Data Flow

### 1. Customer Audio Input

**Entry Point**: `backend/routers/ai_pipeline.py` - `/stt/customer-transcribe` endpoint

When a customer speaks (via the customer panel):
1. Audio bytes are received via WebSocket or HTTP upload
2. Language code is detected or specified (supports 10 Indian languages: hi, mr, ta, te, bn, kn, or, pa, gu, ml)

### 2. Speech-to-Text (STT) Pipeline

**Location**: `backend/services/ai_service.py` - `transcribe()` method

The system uses a **fallback chain** for robust transcription:

1. **Primary**: Sarvam Saarika 2.5 (Indian language optimized)
2. **Fallback 1**: Groq Whisper Large-v3-Turbo
3. **Fallback 2**: Reverie RevUp BFSI (banking-domain trained)

Each step:
- Converts audio to 16kHz mono WAV if needed
- Calls the STT API
- Checks confidence (≥0.6 threshold)
- Falls back if confidence too low or API fails

**PII Detection**: After successful transcription, `pii_service` detects and masks sensitive data (Aadhaar, PAN, phone numbers, etc.)

### 3. LLM Processing

**Location**: `backend/services/ai_service.py` - `process_with_llm()` method

The transcribed text is sent to Groq Llama-3.3-70B (with Gemini backup) with a comprehensive system prompt that:

**System Prompt Features** (`_build_system_prompt()`):
- Translates customer speech to Hindi
- Detects intent from: account_opening, loan_enquiry, kyc_update, card_services, balance_enquiry, fixed_deposit, general
- Analyzes sentiment: calm, frustrated, confused, urgent
- Generates a natural Hindi response for staff to speak
- Translates response to customer's language
- Extracts structured information based on intent schema

**Intent-Specific Schemas**:
- `loan_enquiry`: loan_type, amount, tenure, monthly_income, employment_type, cibil_score, etc.
- `account_opening`: customer_name, account_type, pmjdy_eligible, initial_deposit, etc.
- Each intent has defined fields for information extraction

**Session Context Awareness**:
- Previous conversation history is included (last 6 turns)
- Previously collected info is persisted and merged
- Document readiness is computed and injected into prompts

### 4. Pipeline Orchestration

**Location**: `backend/services/pipeline_orchestrator.py` - `run_transcription_pipeline()`

The orchestrator:
1. Runs STT via AI service
2. Creates database exchange record
3. Builds conversation history with collected info context
4. Pre-detects intent from keywords
5. Computes document readiness gaps
6. Injects session state context (document readiness, phase, next field to ask)
7. Calls LLM with full context
8. Merges extracted info with previously collected data
9. Broadcasts events via WebSocket

### 5. WebSocket Communication

**Location**: `backend/websocket/manager.py`

The system broadcasts real-time events to staff:

| Event Type | Data | Payload |
|------------|------|---------|
| `transcription_ready` | Staff & Customer | text_original, text_translated, confidence, sentiment, intent, pii_detected |
| `ai_suggestion_ready` | Staff Only | suggested_hindi, suggested_customer_lang, intent, process_triggered |
| `audio_ready` | Customer Only | audio_url, duration_seconds, staff_response |
| `info_board_update` | Staff | collected_info, completion_percent, next_question_hindi |
| `doc_readiness_update` | Staff | score, confirmed, missing, docs array |

### 6. Frontend Integration

**Staff Panel** (`frontend/staff-panel`):

#### WebSocket Hook (`useWebSocket.js`)
- Manages WebSocket connection with exponential backoff (max 5 retries)
- Handles message routing via switch statement
- Updates Zustand store with extracted actions:
  - `updateSuggestion()` - stores AI suggestions
  - `addExchange()` - adds conversation entries
  - `updateInfoBoard()` - updates Customer Intelligence board
  - `updateSentiment()`, `updateIntent()` - update state

#### Suggestion Display (`AISuggestionBox.jsx`)
- Receives `aiSuggestion` from Zustand store
- Displays two suggestion texts:
  - **Hindi suggestion**: For staff to read/speak
  - **Customer language translation**: Auto-translated response
- Shows confidence indicator (low confidence warning if <50%)
- Provides "Use Suggested Response" button
- Provides "Edit Suggestion" modal for customization

#### Session Flow
```
Customer speaks → STT → LLM → WebSocket → Staff Panel
                                    ↓
                            AISuggestionBox displays
                                    ↓
                         Staff clicks "Use" or edits
                                    ↓
                    sendStaffApproved/edited → WebSocket
                                    ↓
                   Backend translates → TTS → Customer audio
```

## Special Features

### Conversation Intelligence
- Tracks collected information per session
- Computes completion percentage
- Determines next question deterministically (not repeated)
- Shows navigator phase (greet, educate, collect, verify, process, close)

### Stale Suggestion Prevention
- Compares LLM suggestion against collected info
- Overrides generic document-asking if all docs confirmed
- Prevents asking "Do you have Aadhaar?" when already confirmed

### Document Readiness (DRV)
- Computes document confirmation score
- Shows checklist to customer for confirmation
- Merges customer-confirmed docs with backend state

### Circuit Breakers
- Groq and Gemini have failure thresholds (3 consecutive failures)
- 30-second cooldown for Groq, 60-second for Gemini
- Graceful fallback or error messaging
